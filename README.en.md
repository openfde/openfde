## Organization Introduction


&emsp;&emsp;OpenFDE is a Linux desktop environment based on AOSP, Waydroid, and LineageOS that can run Android applications. OpenFDE aims to bring a rich ecosystem of applications and games to Linux, provide a consistent user experience between Android and Linux applications, explore a unified and novel approach to Linux desktop application development, and offer a brand-new way to experience the Linux desktop.

## Repository Introduction

OpenFDE [Main Repo](https://github.com/openfde)  [Mirror Repo](https://gitee.com/openfde), is primarily used to store source code projects. The repositories mainly include the following categories:：<br>
&emsp;&emsp;&emsp;&emsp;1. Android program-related repositories: Components of the AOSP source tree <br>
&emsp;&emsp;&emsp;&emsp;2. Linux program-related repositories: Such as fde_ctrl, fde_fs, Waydroid_waydroid, fdeion (kernel module), and several others <br>
&emsp;&emsp;&emsp;&emsp;3. Packaging repository: [https://gitee.com/openfde/make_deb](https://gitee.com/openfde/make_deb) <br>
&emsp;&emsp;&emsp;&emsp;4. Repository for managing AOSP source repositories：[https://gitee.com/openfde/manifests](https://gitee.com/openfde/manifests) <br>

[Documentation Site](https://openfde.atlassian.net/wiki/spaces/OpenFDE/overview?homepageId=2457985)，used to store all user-related documentation, development-related documentation, community-related documentation, and blog posts for the OpenFDE community.。

## Feature Introduction
- AOSP Versions:  Supports both AOSP 11 and AOSP 14. <br>
- Linux Display Protocol: Supports both the Wayland and X11 protocols (via waydroid hwc and X11 hwc respectively). <br>
- Device Passthrough: USB camera, power management (for laptops), Wi-Fi (shared with the host).<br>
- Key Assistant: Uses keyboard to simulate touch input, designed for multi-touch scenarios.
